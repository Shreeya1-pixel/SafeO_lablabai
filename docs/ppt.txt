  The gap:

SLIDE 4 — What SafeO is
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Content:
  SafeO = decision engine + investigation room

  1. Scan any input — form, API body, CRM note, webhook
  2. Score with tiered ML (heuristics → neural → local LLM only if needed)
  3. Block before bad data hits the database
  4. Investigate via 4 Band agents on high-risk decisions
  5. Escalate to Jira for the security team

  Works with: Odoo, Salesforce, SAP hooks, Jira, custom apps — anything POSTing JSON.

Speaker Notes:
  "Odoo is our live demo because it's a real ERP. The integration is one REST
   endpoint — same engine everywhere."

Visual:

SLIDE 5 — No OpenAI. No token meter.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Content:
  ML built to run on your metal

  | Tier | Engine | When | Cloud API? |
  |------|--------|------|------------|
  | 1 | Heuristics + patterns | Every request | No |
  | 2 | DistilBERT (CPU/GPU) | Risk 35–65% band | No |
  | 3 | Local Mistral via vLLM | Gray zone only | No — self-hosted |

  Designed outcomes (pilot framing):
  • ~70–85% of requests never touch Tier 3
  • $0 OpenAI / Anthropic spend on default deployment
  • <50ms Tier 1 for typical payloads
  • 100% investigation logs in your stack + Band

Speaker Notes:
  "Regulated buyers don't want every suspicious field shipped to a US cloud LLM.
   Tier 3 is optional local vLLM on AMD. Demo laptop runs Tier 1+2 on CPU fine."

Visual:
  Funnel: 100% requests → wide Tier 1 → thin Tier 3 (~15%)

SLIDE 6 — Band is the collaboration layer
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Content:
  Not a notification pipe — the workflow itself

  On every BLOCK:
  1. MultilingualAgent — script, normalization, evasion flags
  2. PolicyAgent + ForensicsAgent — parallel on Band
  3. RemediationAgent — reads both, proposes actions
  4. Each step posts structured context (scan_id, patterns, severity)

  Judges should see:
  • Delegation ("normalized payload ready" → policy picks up)
  • Shared state — same scan_id in every message
  • Human loop — analyst in Odoo dashboard and Band

Speaker Notes:
  "Band is where Policy and Forensics coordinate while Remediation waits.
   That's the brief — mid-workflow collaboration, not a final paste."

SLIDE 7 — The four agents
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Content:
  MultilingualAgent
  Latin, Arabic, Urdu, Arabizi, mixed-script. Normalizes before pattern scan.
  "Attackers hide SQL in Urdu filler — we still catch it."

  PolicyAgent
  Jurisdiction rules (UAE PDPL, GDPR-style, internal SEC policy).
  Outputs: violated policies, severity, disposition.

  ForensicsAgent
  Attack class: SQLi, XSS, prompt injection, path traversal.
  Outputs: tags, confidence, related patterns.

  RemediationAgent
  Ops steps: block user, rotate creds, notify CISO, open ticket.
  Outputs: ordered action list with owner.

Speaker Notes:
  "Four handles on Band. On block, watch all four post. Policy and Forensics
   overlap on purpose."

Visual:
  Four cards · agent name · one-line job · Band handle

SLIDE 10 — Tiered ML by the numbers
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Content:
  Fast and cheap by design

  • Tier 1 decisive: score < 35% or > 65% — no neural model
  • Tier 2 gray band 35–65%: DistilBERT, blend 40% heuristic + 60% neural
  • Tier 3: only when Tier 2 can't resolve AND local vLLM is up
  • Behavioral layer: velocity, odd hours, repeat probes (+5–15% boost)
  • Drift detection: flags shift vs 7-day baseline
  • Multilingual: AraBERT-class — no external translation API

  Pilot claim (measure in your env):
  ~75–90% of traffic never touches Tier 3
  100% avoids commercial LLM APIs in default deployment

Speaker Notes:
  "Where's the AI? Tier 2 DistilBERT and optional local Mistral. Not a $20k
   monthly OpenAI invoice."

Visual:
